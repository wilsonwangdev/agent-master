# Rules

Agent rules and constraints for this project.

## Quality Gate

A rule can only be added here when it meets ALL of the following:

1. Addresses a verified, recurring issue or enforces a project-wide standard
2. Is actionable and unambiguous — an agent can follow it without interpretation
3. Has been validated through actual project usage
4. Does not conflict with existing rules or tool defaults

## Format

Each rule is a single Markdown file with: the rule statement, rationale, and examples of correct/incorrect behavior.
