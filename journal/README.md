# Journal

Failure records and agent evolution log.

This directory captures incidents, unexpected behaviors, and lessons learned during agent-driven development of this project. Each entry documents what went wrong, why, and what was changed to prevent recurrence.

## Purpose

Enable agent self-evolution by maintaining a structured record of failures and their resolutions. Future agents (and humans) can reference this log to avoid repeating mistakes and to understand the project's evolution.

## Format

Each entry is a Markdown file named `YYYY-MM-DD-brief-description.md` with:
- What happened
- Root cause
- Resolution
- Prevention measures applied
